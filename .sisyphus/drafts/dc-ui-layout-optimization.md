# Dual Contouring Demo UI布局优化工作计划

## 用户确认的设计方案

1. **参数面板布局**: 标签页式布局 - 使用标签页切换不同参数面板
2. **分割器**: 需要可拖动分割线 - 用户可以调整参数与绘制区域的比例
3. **折叠功能**: 参数面板可以折叠展开

## 发现的问题

### 1. 窗口无法拖动
- **原因**: 所有窗口都使用了 `ImGuiWindowFlags_NoMove` 标志
- **代码位置**: main.cpp 第231-235行, 第250-254行, 第280-283行, 第300-304行
- **影响**: 用户无法移动窗口到想要的位置

### 2. 多个参数页面重叠
- **原因**: 控制面板、形状编辑器、统计面板都设置在相同或重叠的位置
  - 控制面板: `(0, 0)` 位置，宽度320px
  - 形状编辑器: `(0, 0)` 位置，宽度300px
  - 统计信息面板: `(0, 420)` 位置
  - 分步控制面板: `(0, 740)` 位置
- **代码位置**: main.cpp 第226-288行
- **影响**: 界面混乱，用户无法同时看到所有面板

### 3. 无法调整参数与绘制区域比例
- **原因**: 没有实现分割器（splitter）
- **当前布局**: 左侧固定320px参数区域，右侧剩余空间为绘制区域
- **影响**: 用户无法根据需要调整两侧的宽度比例

## 目标

1. 实现标签页式参数面板（算法参数、可视化选项、形状编辑器、统计信息分页显示）
2. 实现可拖动分割线，允许用户调整参数区域和绘制区域的宽度比例
3. 允许参数面板折叠，以便在需要时最大化绘制区域
4. 移除 `NoMove` 标志，允许窗口在必要时自由移动

## 需要修改的文件

1. **main.cpp** - 主布局和分割器实现
2. **dc_app.cpp** - 修改面板渲染函数以支持标签页布局
3. **dc_app.h** - 可能需要添加新的状态变量（如当前标签页索引、分割比例等）

## 调研完成

- ImGui分割器实现参考：imgui_demo.cpp中有官方分割器示例
- 标签页实现：可使用ImGui::BeginTabBar + ImGui::BeginTabItem实现
- 折叠功能：使用ImGui::Checkbox或自定义折叠按钮实现
