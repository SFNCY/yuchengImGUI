# AGENTS.md - backends/

## PROJECT: yuchengImGUI - Backend Implementations

## OVERVIEW

Platform and renderer backend implementations for Dear ImGui. Each backend pairs a platform (Win32, GLFW, SDL2, etc.) with a renderer (DirectX, OpenGL, Vulkan, Metal, etc.). Total: 47 files across 20+ distinct implementations.

## CONVENTIONS

| Aspect | Rule |
|--------|------|
| Naming | `{platform}_impl_{renderer}.cpp` e.g. `imgui_impl_glfw_opengl3.cpp` |
| Initialization | Every backend has `ImGui_ImplXXX_Init()` / `ImGui_ImplXXX_Shutdown()` |
| Render loop | `ImGui_ImplXXX_NewFrame()` called before `ImGui::Render()` |
| Shutdown order | Platform shutdown BEFORE renderer shutdown |
| No static state | All context in `ImGuiIO::UserData` |
| Vulkan | Only compiled on non-Apple platforms (CMakeLists.txt condition) |

## ANTI-PATTERNS

- Do NOT call renderer shutdown before platform shutdown (order matters)
- Do NOT store ImGui draw data - render immediately
- Do NOT assume single window - each viewport needs its own pipeline
- Vulkan: Do NOT call `vkDeviceWaitIdle()` during frame rendering

## UNIQUE STYLES

| Backend | Special Note |
|---------|--------------|
| OpenGL3 loader | `imgui_impl_opengl3_loader.h` is auto-generated by glslangValidator |
| Vulkan shader | SPIR-V compiled from GLSL via `backends/vulkan/generate_spv.sh` |
| SDL renderer | Uses `SDL_RenderGeometryRaw()` not OpenGL |
| WGPU | Experimental, maps to D3D12/Metal/Vulkan backends internally |
